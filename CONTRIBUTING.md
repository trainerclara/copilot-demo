
# Contributing

## Branching strategy
- Create feature branches from `main`, e.g., `feat/add-mean-function` or `fix/empty-list-bug`.
- Keep PRs small and focused.

## Commit messages
- Use clear, imperative style: `add mean() with empty-list guard`.
- Reference issues with `Fixes #123` when applicable.

## Pull Requests
Include:
- Summary of change
- How to test
- Screenshots (if applicable)
- Checklist (tests pass, docs updated)
